# From Hard to Soft Clustering

<div class="grid grid-cols-[5fr_4fr] gap-4">
<div>

* **k-Means** assigns each point to exactly **one** cluster → **hard clustering**
	* What if a point is ambiguous — near two cluster centers?
* **Gaussian Mixture Model (GMM)** assigns **probabilities** → **soft clustering**
	* Each point gets a probability of belonging to each cluster
	* Much richer representation!

<br>

> *"GMMs are one of the most elegant examples of the EM algorithm in action"*
> — Andrew Ng

# Mixture Models: Core Idea

* A **mixture model** combines multiple simple distributions into a complex one:

$$p(x) = \sum_{r=1}^R \pi_r \, f_r(x), \quad \text{where} \quad \sum_{r=1}^R \pi_r = 1$$

* $\pi_r$ = **mixing weight** (how prevalent is cluster $r$?)
* $f_r(x)$ = component distribution (e.g. Gaussian)

<br>

<div class="grid grid-cols-[5fr_4fr] gap-4">
<div>

* **Intuition** (Josh Starmer): imagine the data was generated by $R$ different Gaussian "factories"
	* Each factory has its own center (mean) and spread (covariance)
	* We observe the mixed output and want to reverse-engineer which factory made which point
</div>

# The EM Algorithm for GMMs

* We want to estimate the parameters of $R$ Gaussians: $\{\pi_r, \mu_r, \Sigma_r\}_{r=1}^R$
* **Problem**: we don't know which points belong to which Gaussian (latent/hidden variables)
* **Solution**: Expectation-Maximization (EM) — iteratively alternate between two steps:

<div class="bg-orange-100">

**E-step (Expectation)**: compute **responsibilities** — how much does each Gaussian "own" each point?
* $\gamma_{ir} = \frac{\pi_r \, \mathcal{N}(x_i \mid \mu_r, \Sigma_r)}{\sum_{j=1}^R \pi_j \, \mathcal{N}(x_i \mid \mu_j, \Sigma_j)}$ — the soft assignment of point $x_i$ to cluster $r$

**M-step (Maximization)**: update parameters using weighted statistics
* $\mu_r \leftarrow$ weighted mean of all points (weighted by responsibilities)
* $\Sigma_r \leftarrow$ weighted covariance; $\pi_r \leftarrow$ fraction of total responsibility

</div>

* **Analogy** (Andrew Ng): E-step = "guess who made what"; M-step = "given guesses, update factory parameters"

<v-click>

* EM is guaranteed to increase the likelihood at each step (converges to a local maximum)

</v-click>

<!--

# GMM: Practical Considerations

<div class="grid grid-cols-[4fr_4fr] gap-1">
<div>

### When to use GMM
* Clusters have **different shapes/sizes**<br> (not just spherical)
* You need **soft assignments** (uncertainty in cluster membership)
* Data is roughly Gaussian in each cluster
* Feature dimensionality is moderate

<br>

### Choosing $R$ (number of components)
* **BIC** or **AIC** — penalize model complexity
* Cross-validation on downstream task
* Elbow method on log-likelihood
</div>
<div>

### Pitfalls
* ⚠️ Sensitive to initialization (like k-Means)
* Use k-Means to initialize $\mu_r$
* ⚠️ Can **overfit** with too many components
* Covariance can become singular → regularize $\Sigma_r$
* ⚠️ Doesn't scale well to high dimensions
	* Consider diagonal or tied covariance

<br>

### GMM vs. k-Means: Rule of thumb
* Start with k-Means (fast, simple)
* Switch to GMM when clusters are clearly non-spherical or you need probabilities
</div>
